GCP Deployment Walkthrough

This project is designed to run in a lightweight Docker container, making it a perfect fit for serverless hosting on Google Cloud Run.

Prerequisites & Setup

1

Install Tools

Ensure you have the Google Cloud SDK (gcloud CLI) and Docker installed locally.

2

Authorize Google Cloud

Login and configure your active project:

bash
gcloud auth login
gcloud config set project [YOUR_GCP_PROJECT_ID]
3

Enable Required APIs

Activate Cloud Run and Artifact Registry APIs for your project:

bash
gcloud services enable run.googleapis.com artifactregistry.googleapis.com
4

Create Artifact Registry Repo

Create a Docker repository in your preferred GCP region (e.g., us-central1):

bash
gcloud artifacts repositories create ecotrace-repo \
--repository-format=docker \
--location=us-central1 \
--description="EcoTrace Container Repository"
5

Configure Docker Authentication

Authenticate Docker to access Artifact Registry:

bash
gcloud auth configure-docker us-central1-docker.pkg.dev
6

Build and Push the Container

Build the container for the linux/amd64 architecture and push to GCP:

bash
# Build the image using the local Dockerfile
docker build -t us-central1-docker.pkg.dev/[YOUR_PROJECT_ID]/ecotrace-repo/ecotrace:v1 .

# Push the image to Artifact Registry
docker push us-central1-docker.pkg.dev/[YOUR_PROJECT_ID]/ecotrace-repo/ecotrace:v1
7

Deploy to Cloud Run

Deploy the pushed image as a public serverless service on Cloud Run:

bash
gcloud run deploy ecotrace \
--image=us-central1-docker.pkg.dev/[YOUR_PROJECT_ID]/ecotrace-repo/ecotrace:v1 \
--platform=managed \
--region=us-central1 \
--allow-unauthenticated

★ Once deployed, GCP will output a public HTTPS service URL where your application can be accessed globally!
